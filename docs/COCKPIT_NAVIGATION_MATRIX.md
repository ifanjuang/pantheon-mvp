# Cockpit navigation hierarchy

Status: UX contract aligned with the five-space information architecture and reconciled with current `main` capability-management posture.

| Level 1 | Level 2 | Level 3 / cards | Purpose |
|---|---|---|---|
| Pantheon | Context | Pantheon Context | Active professional context, attention and boundaries |
| Pantheon | Attention | Work review, Knowledge review, partial Document | Cross-space items needing examination |
| Affaires | Affaire active | Project Card | Project identity and access to project projections |
| Affaires | Répertoires | Intervenants & contacts; Entreprises | Project-scoped people and company directories |
| Affaires | Dossiers | 00_Gestion; 10_Conception; 20_Autorisations; 30_DCE; 40_Marche; 50_Chantier; 60_Reception; 90_Sinistres | Logical navigation over Project Documents |
| Affaires | Travail | Work Issue | Project work, review state and Hermes traces |
| Affaires | Documents | Project Document Card | Source-backed project documents and extraction |
| Affaires | Knowledge liée | Knowledge Item | Reusable knowledge linked to the project |
| Connaissances | Dossiers Knowledge | one folder per current family | Logical reusable-knowledge navigation |
| Connaissances | Dossier ouvert | Knowledge Item | Items within the selected family |
| Outils | Capacités | Lifecycle + Hermes executor card | Governed capability lifecycle posture |
| Outils | Runtime documentaire | Candidate observation card | Paperless/PDP/Docling/Hermes observations when adopted |
| Décisions | Inbox | Decision Request / Gate | Human review requests derived from Work Issues in review |

`Documents`, `Travail`, `Questionnaire`, Resource Profiles and proposal-only effects are projections/actions inside these spaces, not additional level-1 navigation.
